<AccordionGroup>
  <Accordion title="What is Herald Protocol?">
    Herald Protocol is the infrastructure for the agentic economy on 0G Chain. It ships an x402 facilitator so any API, MCP server, or agent can accept onchain payments on 0G, an MPP SDK for direct server-side settlement without a facilitator, and a cross-chain payment router so agents holding funds on 0G can pay services on other chains without managing multiple wallets.
  </Accordion>

  <Accordion title="What is x402?">
    [x402](https://x402.org) is an open payment standard built around the HTTP `402 Payment Required` status code. A server returns payment requirements; a client signs a payment; a facilitator verifies and settles it onchain. See the [x402 docs](https://docs.x402.org) for the full protocol.
  </Accordion>

  <Accordion title="Do I need to run my own facilitator?">
    No. Point your server at `https://facilitator.heraldprotocol.xyz` and you're done. The facilitator is free to use and drop-in compatible with the x402 spec.
  </Accordion>

  <Accordion title="Which networks are supported for the facilitator?">
    0G Mainnet (`eip155:16661`) and 0G Galileo Testnet (`eip155:16602`). See [Supported networks](/x402/supported-networks).
  </Accordion>

  <Accordion title="Which tokens can I accept?">
    Bridged USDC (`USDC.e`) on 0G Mainnet, which you can swap for via [hub.0g.ai](https://hub.0g.ai/swap?network=mainnet). 0G testnet has no official stablecoin yet — a Herald Protocol EIP-3009-compatible test token is on the roadmap.
  </Accordion>

  <Accordion title="Does the facilitator hold funds?">
    No. The facilitator verifies signed payment payloads and submits them to 0G. It never custodies funds. Settlement goes directly from buyer to the seller's `payTo` address onchain.
  </Accordion>

  <Accordion title="Can I use any x402 or MPP SDK?">
    Yes. The facilitator implements the standard `/verify`, `/settle`, and `/supported` endpoints, so any x402-compatible client or server library works. On the seller side, `@heraldprotocol/x402` wraps `@x402/evm` with 0G-aware money parsing so you can price endpoints in USD. For MPP, `@heraldprotocol/mpp` provides native 0G support with direct server-side settlement — no facilitator required.
  </Accordion>

  <Accordion title="What is the Herald Router?">
    The Herald Router is a hosted proxy at `https://router.heraldprotocol.xyz` that lets agents holding funds on 0G pay any x402-protected service on another supported chain. The agent pays the router on 0G; the router pays the destination using its own wallet on the target chain. No bridging, no multi-chain wallet management. See [Router Overview](/router/overview).
  </Accordion>

  <Accordion title="Which chains does the router support?">
    Currently 0G Mainnet (`eip155:16661`) and Base Mainnet (`eip155:8453`). The agent always pays from 0G; the router settles on whichever chain the destination service requires.
  </Accordion>

  <Accordion title="Does the router hold funds?">
    The router holds pre-funded wallets on each supported destination chain to settle outbound payments. Your agent sends payment to the router on 0G; the router bridges that value by paying the destination on your behalf from its own wallet.
  </Accordion>

  <Accordion title="What's on the roadmap?">
    - [x] **MPP SDK for 0G** — Native MPP payment method with `@heraldprotocol/mpp` for direct server-side settlement
    - [x] **Accept any ERC-20 as payment** — supports `Permit2` gas sponsoring to enable any ERC-20 payments
    - [ ] **More router destination chains** — Arbitrum, Solana, Stellar and beyond
    - [ ] **MPP on Router** — Route MPP payments through the Herald Router. Agents pay on 0G; the router settles via Tempo, Stripe, card, or other fiat rails on the destination side.
    - [ ] **x402 Bazaar and ERC8004** — Discovery layer for trustless agent services on 0G Chain.
  </Accordion>
</AccordionGroup>
