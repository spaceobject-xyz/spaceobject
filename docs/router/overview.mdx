The Herald Router is a cross-chain payment router. It lets agents holding funds on 0G pay services on any supported chain — without bridging manually or maintaining wallets on multiple networks.

```
https://router.heraldprotocol.xyz
```

Hosted endpoint. One request routes payment from 0G to any supported destination chain.

## The problem it solves

Payment-protected services exist on many chains. An agent that only holds USDC.e on 0G cannot directly pay a service that settles on Base. Bridging manually means juggling multiple wallets, funding them with native tokens for gas, and managing cross-chain state — none of which belongs in an agent's payment logic.

The Herald Router handles all of that transparently. Your agent makes one call; the router handles the rest.

| Protocol | Endpoint | Status |

## How it works

<Steps>
  <Step title="Agent requests a resource via the router">
    Instead of calling the destination URL directly, the agent calls `GET /route/x402?url=<destination>` on the router.
  </Step>
  <Step title="Router proxies the request">
    The router forwards the request to the destination. If the response is `200 OK`, it passes the response back immediately — no payment needed.
  </Step>
  <Step title="Router reads the payment challenge and re-presents it">
    If the destination requires payment, the router reads the payment requirements and re-presents them to the agent mapped to supported networks. The agent sees 0G as the payment option regardless of where the destination settles.
  </Step>
  <Step title="Agent pays the router on 0G">
    The agent signs a payment authorization on 0G (USDC.e, `eip155:16661`) and retries the request with the payment header, exactly like any other payment flow.
  </Step>
  <Step title="Router pays the destination on the target chain">
    Once the router verifies and settles the inbound payment, it calls the destination using its own wallet on the target chain and returns the response to the agent.
  </Step>
</Steps>

## What the agent needs

- A funded wallet on **0G Mainnet** (USDC.e — swap via [hub.0g.ai](https://hub.0g.ai/swap?network=mainnet))
- The client stack for your protocol — e.g. for x402: `@x402/core`, `@x402/evm`, `@x402/fetch`
- The router URL for your protocol — e.g. for x402: `https://router.heraldprotocol.xyz/route/x402?url=<destination>`

The agent never needs wallets on destination chains. The router abstracts that entirely.

## What the router charges

The router applies the destination's payment requirements directly — no additional fee is added on top of the service price. The agent pays exactly what the destination service charges, denominated in 0G USDC.e.
