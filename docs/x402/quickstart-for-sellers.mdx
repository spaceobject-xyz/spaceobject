This guide walks you through putting an x402 paywall in front of an HTTP endpoint using the Herald Protocol facilitator. The example uses [Hono](https://hono.dev), but the same pattern works with any x402 server adapter (Express, Next.js, Fastify, etc.).

## Prerequisites

- A 0G-compatible EVM wallet address to receive payments
- Node.js 18+ (or any runtime with WebCrypto and `fetch`)
- A server framework supported by x402 — this guide uses Hono

## 1. Install dependencies

<CodeGroup>

```bash npm
npm install @heraldprotocol/x402 @x402/core @x402/evm @x402/hono hono
```

```bash pnpm
pnpm add @heraldprotocol/x402 @x402/core @x402/evm @x402/hono hono
```

```bash bun
bun add @heraldprotocol/x402 @x402/core @x402/evm @x402/hono hono
```

</CodeGroup>

## 2. Point at the Herald Protocol facilitator

Create an `HTTPFacilitatorClient` targeting `https://facilitator.heraldprotocol.xyz`. This client handles the `/verify` and `/settle` round trips on your behalf.

```ts server.ts
import { HTTPFacilitatorClient } from "@x402/core/server";
import {
  declareEip2612GasSponsoringExtension,
  declareErc20ApprovalGasSponsoringExtension,
} from "@x402/extensions";
import { paymentMiddleware, x402ResourceServer } from "@x402/hono";
import { Hono } from "hono";
import { logger } from "hono/logger";

import { ExactEvmScheme } from "@heraldprotocol/x402/server";

const app = new Hono();
const evmAddress = "0xYourEvmAddress"; // where payments settle

const facilitatorClient = new HTTPFacilitatorClient({
  url: "https://facilitator.heraldprotocol.xyz",
});

app.use(logger());
```

## 3. Register the 0G scheme and protect a route

`@heraldprotocol/x402/server` exports an `ExactEvmScheme` pre-configured for 0G's default stablecoin. It resolves USD-style prices (e.g. `"$0.001"`) to the correct token amount automatically.

```ts server.ts
app.use(
  paymentMiddleware(
    {
      "GET /weather": {
        accepts: [
          {
            scheme: "exact",
            price: "$0.001",
            network: "eip155:16661",
            payTo: evmAddress,
          },
        ],
        description: "Weather data",
        mimeType: "application/json",
        extensions: {
          ...declareEip2612GasSponsoringExtension(), // Enable EIP-2612 gas sponsoring
          ...declareErc20ApprovalGasSponsoringExtension(), // Enable ERC-20 approval gas sponsoring
        },
      },
    },
    new x402ResourceServer(facilitatorClient).register(
      "eip155:16661",
      new ExactEvmScheme()
    )
  )
);

app.get("/weather", (c) =>
  c.json({ report: { weather: "sunny", temperature: 70 } })
);

export default app;
```

<Note>
  Use `"eip155:16602"` instead of `"eip155:16661"` to target 0G Galileo Testnet.
</Note>

## 4. Test it

Start your server and hit the protected route without a payment:

```bash
curl -i http://localhost:3000/weather
```

You should see a `402 Payment Required` response with the `accepts` list in the body. Any x402-compatible client can now pay this endpoint — see [Quickstart for Buyers](/x402/quickstart-for-buyers).

## Route configuration reference

Each protected route declares one or more `accepts` entries. A client picks the first entry it supports.

| Field | Description |
| --- | --- |
| `scheme` | `"exact"` for a fixed price. |
| `price` | USD-style string (e.g. `"$0.001"`) or an atomic token amount as a string. |
| `network` | CAIP-2 network ID — `eip155:16661` (mainnet) or `eip155:16602` (testnet). |
| `payTo` | EVM address that receives the payment. |

## Gas sponsoring extensions

To support **any ERC-20 token** (not just EIP-3009 tokens like USDC.e), advertise these extensions in your route configuration:

| Extension | Purpose |
| --- | --- |
| `declareEip2612GasSponsoringExtension()` | Gasless EIP-2612 permit for tokens that support it (preferred). |
| `declareErc20ApprovalGasSponsoringExtension()` | Gasless raw ERC-20 `approve()` for tokens without EIP-2612 (fallback). |

The client automatically picks the best option: EIP-2612 if the token supports it, ERC-20 approval otherwise. Both are handled transparently — no code changes needed on the client beyond providing `signTransaction` and `getTransactionCount` capabilities.

## Using another framework

The only Herald Protocol-specific wiring is the `ExactEvmScheme` import and the `HTTPFacilitatorClient` URL. The rest is standard x402 — swap `@x402/hono` for `@x402/express`, `@x402/next`, or another adapter and keep the same middleware contract.
